# Contributing to TensorMap

Thank you for contributing to TensorMap! Please follow these guidelines to keep our development process smooth and organized.

## Branch Naming Conventions
- Features: `feat/phase-N-description`
- Bug Fixes: `fix/description`
- Testing: `test/description`

## Commit Style
Please follow [Conventional Commits](https://www.conventionalcommits.org/). Use prefixes:
- `feat:` for new features
- `fix:` for bug fixes
- `test:` for adding or updating tests
- `docs:` for documentation updates
- `refactor:` for code changes that neither fix a bug nor add a feature

## PR Size Guideline
Keep Pull Requests focused and manageable. Try to keep PRs under **800 lines of code**. If a PR exceeds this size, consider breaking it up into smaller, sequential PRs.

## Review Cadence
PRs and overall progress are evaluated during **mentor reviews scheduled at the end of each week**. Please aim to have your PRs ready and passing CI by then.
